# Live-Ops and Content Cadence

## Cadence Baseline
- Biweekly event cycle (14 days).
- Monthly seasonal layer for broader progression framing.

## Biweekly Event Template
Week 1:
- Event launch objectives.
- Increased activity participation incentives.

Week 2:
- Catch-up objectives.
- Final milestone push and reward claim window.

## Monthly Seasonal Structure
- Season theme and cosmetic line.
- 3-5 long-form objectives.
- Limited-time shop stock and token sink.

## Live-Ops Configuration Surface
- Event start/end timestamps.
- Reward table IDs.
- Progress objective IDs.
- Currency exchange rates (event token conversions).

## Content Production Pipeline (Target)
1. Design drafts event goals and reward tables (T-3 weeks).
2. Balance review and economy simulation pass (T-2 weeks).
3. UI/art asset lock and integration prep (T-10 days).
4. QA and telemetry checklist (T-5 days).
5. Controlled rollout to internal/beta (T-2 days).
6. Global activation and monitoring (launch day).

## Safety Rails
- Hard caps on event currency generation per day.
- Server-configured emergency tunables:
  - reward multiplier
  - objective threshold
  - shop pricing coefficients
- Rollback plan for broken objectives or reward exploits.

## Metrics to Track
- Event participation rate.
- Completion funnel by objective tier.
- Currency source/sink balance during event.
- Retention delta during and after event windows.
